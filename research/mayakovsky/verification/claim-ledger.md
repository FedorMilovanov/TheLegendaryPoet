# Реестр утверждений — Маяковский и Брики

Рабочий реестр для переноса исследовательских данных в статьи.

## Правила

- `READY` не означает, что формулировку можно копировать из книги дословно. Это означает, что факт достаточно опорный для редакционного пересказа с источником.
- `VERIFY` требует поиска первичного или независимого источника.
- `DISPUTED` требует явной маркировки конкурирующих версий в статье.
- `HOLD` не должен попадать в публикацию до проверки.

## Следующий проход

Для каждого `VERIFY`:

1. найти первоисточник;
2. записать стабильный URL или полную библиографию;
3. добавить архив/учреждение;
4. уточнить точную дату;
5. переписать утверждение без лишней психологизации;
6. сменить статус на `READY` либо `DISPUTED`;
7. указать конкретный блок статьи.
