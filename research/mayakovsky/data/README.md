# Машиночитаемые данные переписки Маяковского и Бриков

## `correspondence-index-1-416.csv.gz`

Сжатый metadata-only индекс всех 416 нумерованных единиц из репринта переписки, подготовленного Бенгтом Янгфельдтом.

Файл **не содержит текстов писем, телеграмм или комментариев**. Он хранит только библиографические и навигационные поля, необходимые для:

- проверки чисел и дат в статьях;
- построения будущей временной шкалы;
- фильтров по отправителю, адресату, жанру и маршруту;
- визуализации географии переписки;
- поиска единицы по номеру и печатной странице;
- контроля редакционных и источниковедческих оговорок.

## Поля

| Поле | Значение |
|---|---|
| `number` | номер единицы в издании, строго 1–416 |
| `print_page` | печатная страница репринта, на которой начинается единица |
| `sender` | буквальная левая часть заголовка: `ВВМ`, `ЛЮБ`, `ОМБ` и совместные подписи |
| `recipient` | буквальная правая часть заголовка |
| `date` | редакторская или документальная дата в форме издания после осторожной OCR-нормализации |
| `route` | город или направление отправки; дефис между городами нормализован в тире |
| `document_type` | `telegram`, `letter`, `note`, `postcard` или `inscription` |
| `provenance_code` | краткий код источникового статуса текста |

## Коды происхождения

| Код | Рабочее значение |
|---|---|
| `M-PHOTO-VERIFY` | документ с Маяковским среди отправителей; общий текстологический комментарий указывает преобладание фотокопий, но конкретную единицу ещё нужно проверять отдельно |
| `B-TYPE-VERIFY` | документ Лили или Осипа; общий слой машинописных копий, оригиналов и фотокопий требует поединичной проверки |
| `LILI-TYPE-EXACT` | текст напечатан по машинописному списку Лили, заявленному составителем как точное воспроизведение подлинника |
| `UNAUTH-TYPE` | неавторизованная машинописная копия |
| `LILI-1956-INCOMPLETE` | № 113: версия Лили 1956 года при неполно сохранившемся подлиннике |
| `OTHER-VERIFY` | иной корреспондент; основание текста проверяется по конкретной единице |

Код происхождения не является архивным шифром и не заменяет чтения комментария, автографа или описи.

## Воспроизводимость

Индекс заново строится из локальной копии PDF:

```bash
python3 scripts/build-mayakovsky-correspondence-index.py \
  /path/to/jangfeldt-correspondence-1991.pdf \
  --output /tmp/correspondence-index-1-416.csv

gzip -9 /tmp/correspondence-index-1-416.csv
```

Скрипт требует установленный `pdftotext`, извлекает заголовки и комментарии, применяет ограниченный список проверенных OCR-нормализаций и завершает работу ошибкой, если:

- отсутствует хотя бы один номер;
- нарушена последовательность 1–416;
- изменились суммы жанров или годов;
- пусто обязательное поле;
- не совпали контрольные единицы № 1, 113, 397 и 416.

CI дополнительно проверяет уже сохранённый `csv.gz`, поэтому генератор и опубликованный массив контролируются независимо.

## Ограничения

- Даты вроде `Весна 1923 г. (?)` сохраняют редакторскую неопределённость.
- `sender` и `recipient` следуют заголовкам издания и не устанавливают автора каждой строки совместного документа.
- Жанр является рабочей классификацией формы опубликованной единицы, а не обязательным архивным названием.
- Общая арифметика издания `217 документов Маяковского / 194 документа Лили / 5 телеграмм Осипа` не подменяет буквальный подсчёт заголовков `218 / 193 / 5`; расхождение одной единицы документировано отдельно.
- Для публичной интерактивной страницы данные следует преобразовать в JSON на этапе сборки, не загружая gzip и исследовательский слой непосредственно в браузер.
