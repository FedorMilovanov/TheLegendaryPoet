## Найденные проблемы

### 1. Каталог `originals` содержит не Commons originals

Из 30 файлов:

- 28 скачаны через `images.weserv.nl` как `transparent resize of Commons original`;
- 2 скачаны как `official Commons thumbnail`.

Следовательно, SHA-256 относится к перекодированному JPEG/thumbnail, а не к исходному бинарному файлу Wikimedia Commons. Название каталога `originals` и формула «скачивает оригиналы» неточны.

#### Требование

Скачивать байты непосредственно из `original_url`. Прокси разрешать только как явный fallback, сохраняя два checksum:

- `commons_original_sha256`;
- `render_derivative_sha256`.

### 2. Лицензионный manifest неполон

У всех 30 записей:

- `license_name = Public domain`;
- `license_url` пуст.

Это не позволяет проверить конкретный Commons template и юридическое основание public-domain статуса каждого объекта.

#### Требование

Сохранять:

- `license_short_name`;
- `license_url`;
- исходные Commons templates (`PD-Russia`, `PD-old`, `PD-US`, etc.);
- `credit_line`;
- дату последней проверки карточки.

Если URL отсутствует в API, сохранять stable Commons page URL и исходный wikitext/template, а не пустое поле.

### 3. Повреждены имена авторов

Примеры manifest:

- `Unknown authorUnknown author`;
- `Unknown photographerUnknown photographer`;
- `UnknownUnknown photograph`.

Такие значения появились при склейке HTML/metadata и непригодны для публичных credits.

#### Требование

- нормализовать дублирующиеся текстовые узлы;
- сохранять raw HTML отдельно;
- для неизвестного автора использовать одно каноническое значение `Unknown author` или `Unknown photographer`;
- не превращать отсутствие атрибуции в имя автора.

### 4. `download-audit.txt` фактически пуст

Файл имеет размер 1 байт и не содержит журнал сетевых ответов, fallback или ошибок. Он не подтверждает, что каждая Commons API-карточка была проверена.

#### Требование

Audit должен содержать по каждой позиции:

- HTTP status Commons API;
- exact page id и revision id;
- original URL status;
- fallback status;
- размер полученных байтов;
- MIME type;
- checksum;
- parsed license template;
- warning/error.

### 5. Два объекта заметно меньше остальных

- позиция 9: 396×559 px;
- позиция 11: 434×755 px.

Остальные преимущественно приведены к ширине 1800 px. Для контактного листа этого достаточно, но для самостоятельного крупного показа качество ограничено.

### 6. Даты и атрибуции пока повторяют Commons metadata без независимой проверки

Workflow проверяет наличие metadata, но не доказывает историческую точность каждой подписи. Рисковые карточки:

- `Lilya Brik in 1906.jpg`;
- `Lilya Brik with her mother and sister Elsa.jpg`;
- `Vladimir mayakovsky and lilya brik.jpg`;
- `Mayakovsky Pasternak.jpg` с точной датой `1924-05-11`;
- `Maiakowski 1925.jpg` с диапазоном `November-December 1925`.

Нужна сверка с музейными каталогами, публикациями фотографий или академической фотолетописью. Commons остаётся маршрутом к файлу, но не гарантирует безошибочную историческую атрибуцию.

Публикационный вариант должен быть новым layout, построенным из уже проверенного manifest, а не прямой загрузкой этого PNG.
