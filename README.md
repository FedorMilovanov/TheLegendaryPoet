# THE LEGENDARY POET

Тёмный люксовый редакторский сайт о великих русских поэтах: биографии, тексты,
глубокий литературный и — там, где это оправдано — христианский разбор.
React 19 + TypeScript + Vite + Tailwind v4 + Framer Motion.

## Разработка

```bash
npm install
npm run dev          # локальный сервер разработки
npm run check        # контент, источники, медиа, TypeScript и Playwright
npm run build        # production-сборка в dist/
npm run preview      # предпросмотр собранной версии
npm run images:sync  # локализация архивных оригиналов, AVIF → WebP
npm run sitemap      # пересобрать public/sitemap.xml
npm run prerender    # статические SEO/OG-страницы
```

## Общая архитектура длинных материалов

Новые `/essays/:slug` и прежние `/articles/:id` используют один блоковый `ArticleRenderer`.
Новая статья создаётся как данные (`EssayBlock[]`), а не как новый набор точечных компонентов и CSS.

Общие возможности движка:

- заголовки и автоматически формируемое оглавление;
- проза, цитаты, стихи, примечания и итоговые размышления;
- inline citations со стабильными `sourceIds`;
- единая библиотека источников с разделением на первичные документы, архивы, исследования, институциональные нарративы и контекст;
- изображения `wide`, `portrait` и `cinematic`, включая мягкое обтекание текста;
- общий lightbox, масштабирование, drag, Esc, возврат фокуса и reduced-motion;
- единый локальный медиаманифест, LQIP и responsive AVIF/WebP;
- отдельные поля для происхождения архивного изображения, реставрации или художественной реконструкции.

Основные точки расширения:

- `src/types/essay.ts` — типы статьи, блоков, изображений и источников;
- `src/components/essay/ArticleRenderer.tsx` — общая композиция лонгрида;
- `src/components/essay/blocks.tsx` — один компонент на тип блока;
- `src/components/essay/media.ts` — разрешение локального медиаманифеста;
- `src/data/essays/*` — данные статей, библиографии, citation rules и visual placement;
- `scripts/validate-essays.ts` — редакционные и источниковедческие ограничения;
- `scripts/validate-essay-media.ts` — проверка responsive-наборов;
- `e2e/essay-engine.spec.ts` — полный Playwright-прогон общего движка.

## Изображения: AVIF сначала, WebP затем

Runtime-контракт:

```text
<picture>
  <source type="image/avif" ... />
  <source type="image/webp" ... />
  <img src="...webp" ... />
</picture>
```

`npm run images:sync`:

1. скачивает канонический архивный оригинал один раз;
2. создаёт responsive AVIF-варианты;
3. создаёт совпадающие по ширине WebP-варианты;
4. сохраняет WebP как совместимый `<img>` fallback;
5. генерирует LQIP и `src/generated/essayMedia.ts`.

Chromium Playwright обязан подтвердить, что `currentSrc` реально заканчивается на `.avif`; наличие AVIF-файла без фактического выбора браузером проверку не проходит.

## Исследовательская архитектура

- `docs/RESEARCH_SOURCES.md` — общий список корпусов и редакционная очередь;
- `research/SOURCE_POLICY.md` — обязательная иерархия доказательств;
- `research/<poet-id>/` — книги, досье, claim ledger, визуальный реестр и worklog;
- `src/docs/THEOLOGICAL_GUIDELINES.md` — правила богословской и нравственной оценки.

Авторитетность учреждения не меняет тип материала. Музейная карточка конкретного автографа может быть архивной опорой; музейный биографический очерк или виртуальная выставка остаются институциональным нарративом и не входят в минимум первоисточников.

## Структура приложения

- `src/pages/*` — маршруты `/`, `/poets`, `/poets/:id`, `/articles`, `/articles/:id`, `/essays/:slug`, `/music`, `/about`, 404;
- `src/data/library/*` — профили поэтов, стихи, краткие статьи и музыкальные данные;
- `src/data/essays/*` — большие блоковые статьи и их документальный слой;
- `src/config/site.ts` — единый источник ссылок на каналы и контактов;
- `src/hooks/useSeo.ts` — title, description, OG и canonical;
- `research/*` — непубликуемый исследовательский слой;
- `public/images/essays/*` — локальные web-деривативы, не внешние runtime hotlinks.

## Публикация на GitHub Pages

Всё настроено под project‑страницу `https://<owner>.github.io/TheLegendaryPoet/`:

1. В настройках репозитория: **Settings → Pages → Source: GitHub Actions**.
2. Пуш в `main` запускает `.github/workflows/deploy.yml`.
3. `vite.config.ts` `base` = `/TheLegendaryPoet/`; deep-ссылки работают через `404.html`‑фолбэк.
4. Для кастомного домена задайте `VITE_BASE=/`, обновите `siteConfig.url`, OG-URL и `public/CNAME`.

Готово к публикации и на Netlify/Vercel (`_redirects`, `vercel.json`).

## Редакционные стандарты — обязательно для контрибьюторов и ИИ-агентов

1. Библейский разбор появляется только там, где текст или биография дают основание.
2. Талант, библейская образность и тоска по свету не доказывают личную веру поэта.
3. Источник библейского образа объясняется точно, но автор не объявляется пророком или христианином без свидетельств.
4. Нравственная оценка формулируется после документальной части — прямо, уважительно и без сентиментального обеления.
5. Канонические тексты сверяются минимум по двум независимым изданиям.
6. Биографические тезисы проходят путь `источник → research note → claim ledger → статья → библиография`.
7. Музейный, семейный, советский и обвинительный контрнарративы проверяются по исходным письмам, рукописям и делам.

**Бренд:** эмблема — фигура в капюшоне, анонимный «легендарный поэт». Не возвращать старый логотип `LP` и не ставить христианские ярлыки на главную обложку.

## Публикация и лицензии

Тексты стихотворений используются только после проверки правового статуса и редакции. Для каждой фотографии отдельно устанавливаются автор, дата, коллекция, происхождение и лицензия. Художественная реконструкция всегда маркируется как реконструкция и не выдаётся за архивный снимок.

Лицензию на код и оригинальные редакционные материалы необходимо определить до публичного релиза.
