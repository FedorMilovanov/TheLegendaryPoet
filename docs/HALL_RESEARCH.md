# Зал Поэтов — референсы и исследование

Рабочие материалы по 3D/иммерсивному «Залу Поэтов», чтобы не терялись между сессиями.

## Референсы

- `reference/hall_target_v2.jpg` — первый референс («игровой», неоклассический неф с нишами).
  Под него была сделана текущая версия зала (`/hall`, `src/components/hall/`).
- `reference/hall_target_v3_temple.webp` — новый референс «Храм Русской Поэзии»: купольная
  ротонда-пантеон с центральным атриумом и 4 крыльями по эпохам (Золотой век, Серебряный век,
  Советская поэзия, Современные поэты). Значительно амбициознее v2: резной камень, статуи,
  роспись купола, кессонированный потолок.

## Стек и ключевые технические решения

- React Three Fiber (R3F) v9 + three.js r0.184 + `@react-three/drei` v10 +
  `@react-three/postprocessing` v3 + `postprocessing` (pmndrs) — стандартный премиальный стек
  для WebGL-музейных сцен, не самописный рендер.
- Постпроцессинг: N8AO (ambient occlusion) → selective Bloom → Vignette, ACES Filmic
  tone mapping на рендерере (не как отдельный effect, чтобы не было двойного тонмаппинга).
- **Важный найденный баг класса библиотеки:** React Router context не пробрасывается через
  `<Canvas>` — `useNavigate()` нельзя вызывать внутри Canvas-дерева, только снаружи с
  прокидыванием callback'а вниз. Актуально для любого будущего 3D-компонента с навигацией.
- **Важное ограничение:** Framer Motion `layoutId` не может морфить между DOM и WebGL-канвасом
  (разные реконсиляторы, непрозрачный framebuffer). Морф-переход портрет→страница поэта нужно
  делать через `getBoundingClientRect()` FLIP поверх персистентного канваса, а не через
  `layoutId`.

## Почему текущий зал выглядит плоско/«космически», а не как музей

Материалы зала (`src/components/hall/materials.ts`) уже год как имеют готовый PBR-пайплайн
(albedo/normal/roughness/AO для мрамора и травертина), но `USE_SCANS = false`, и папки
`/public/textures/hall/` физически нет — ни одной настоящей текстуры. Всё стоит на плоских
`meshStandardMaterial color="#242b34"`. Это не баг кода, а отсутствующий актив.

**Бесплатный путь исправления:** CC0 PBR-текстуры с Poly Haven (polyhaven.com/textures) —
мрамор, травертин, латунь — подключаются в уже готовый `useFloorMaterial()`/`useWallMaterial()`
простым переключением `USE_SCANS = true`. Баз запечённого света (`hall_lm_4k.ktx2` и т.п.,
см. комментарий в конце `materials.ts`) можно получить бесплатно через Blender (open source),
экспортировав в KTX2.

## Оценка референса v3 (купольный храм)

- **Форма (купол + 4 крыла)** — это геометрия, не искусство. Строимо на three.js, и прямо
  ложится на концепцию «залы по эпохам».
- **Фотореалистичная резьба, статуи, роспись купола** — это не пишется кодом. Нужны либо
  готовые 3D-ассеты (CC0/покупные), либо запечённый свет + реальные PBR-сканы, либо и то, и
  другое. Пиксель-в-пиксель повторить референс — нереалистично без ручного 3D-моделирования
  (уровень AAA-архвиза). Прийти к духу референса (купол, радиальная симметрия, тёплый свет,
  настоящий камень/золото) — реалистично, поэтапно, без костылей.

## 12.07.2026 — найдена вторая, более развитая версия проекта

Пользователь прислал `rebuildinglegendarypoetreactproject_1.zip` — отдельная сборка того же
проекта (87 файлов), которая никогда не пушилась в GitHub. Сравнение с текущим репо:

**У репозитория (GitHub, эта ветка) сильнее:**
- Контент почти в 2 раза объёмнее (например `sergeiYesenin.ts`: 169 строк против 93 в zip;
  вся сверка фактов и текстов из `docs/RESEARCH_SOURCES.md` — только здесь).
- Логика сообщества/комментариев доработана точнее (`commentHighlights.ts`: посл овный анализ
  отрицаний вместо посимвольного substring-match — меньше ложных срабатываний).

**У zip сильнее:**
- `InteractivePoemText` — тот самый эффект «слова становятся золотыми» при наведении/чтении,
  которого пользователь не находил: dwell-фокус, soft-glow, «след прочитанного» золотом,
  push-эффект соседних слов. **Перенесён в репозиторий** as
  `src/components/poet-detail/InteractivePoemText.tsx`, подключён в `PoemCard.tsx`.
- ~20 доп. компонентов: `LoadingScreen`, `TiltCard`, `KineticText`, `MobileDock`,
  `ThemeToggle`, `AudioPlayer`, `PoetPortrait`/`PoetImage`, `DebugPanel`, `MobileGestureHints`,
  `PoemOfDay`, `PremiumIcons` и др. — не перенесены, следующий этап.
- Более богатая модель поэтов: `epochColors.ts`, `poetConnections.ts` (связи между поэтами:
  влияние/личное/эпоха/полемика), `poetMuseumMeta.ts` (материал памятника, акцентный цвет,
  главная цитата). Эпоха выводится из уже существующих `tags`, так что накладывается на
  текущий контент почти без миграции.
- Другой Зал Поэтов: не неф с нишами, а открытый «музей» с зонами по эпохам, «созвездием»
  поэтов на потолке, анимированными связями-дугами между поэтами, `camera-controls` для
  кинематографичной камеры, шкалой времени (timeline scrubber), режимом тура и режимом «сеть
  связей». **Концептуально интереснее**, но использует звёздное небо/космический фон — то есть
  прямо противоречит решению «музей, не космос» — и тоже не похож на референс v3 (купол).

**Вывод:** обе версии Зала Поэтов не подходят в исходном виде. План — Hall v3: взять
структуру данных и идеи zip-версии (эпохи, связи между поэтами, timeline) и выразить их в
музейной мраморной эстетике текущей версии (крылья-ниши по эпохам вместо звёздного неба),
двигаясь поэтапно к референсу v3 (купол).
