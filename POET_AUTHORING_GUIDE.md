# POET_AUTHORING_GUIDE.md — добавление и унификация поэтов

Последний раз обновлено: 2026-07-21. Владелец: Фёдор Милованов.

Это обязательный рабочий стандарт для новых и существующих страниц поэтов THE LEGENDARY POET. Его задача — сохранить единую архитектуру, точность источников и живой русский голос без школьного глянца, сенсационности и машинной риторики. Осознанное отступление фиксируется в PR.

> - `PROJECT_CHARTER.md` — позиция, цели и иерархия требований.
> - `docs/EDITORIAL_JUDGMENT_AND_SOURCE_POLICY.md` — праведное суждение и источники.
> - `docs/HISTORICAL_NARRATIVE_STANDARD.md` — история прежде вывода и русский стиль.
> - `src/docs/THEOLOGICAL_GUIDELINES.md` — богословская оптика.
> - `AGENT_ANTI_REGRESSION_PROTOCOL.md` — проверки и защита от откатов.
> - `ARENA_PROJECT_PROTOCOL.md` — архитектура и дизайн.

## 0. Редакционная позиция

Проект ведёт Фёдор Милованов. Богословская позиция — историческое евангельское христианство: богодухновенность и достаточность Писания, спасение по благодати через веру во Христа, грамматико-историческое чтение Библии.

Эта позиция задаёт нравственную оптику, но не превращает каждую биографию в проповедь. Основной корпус страницы — история, документы, стихи и литературный анализ. Грех не замалчивается, поэт не объявляется святым задним числом, а библейская оценка появляется там, где её подготовили факты.

## 1. Архитектура: один поэт — один модуль

```text
src/data/library/
  <poetName>.ts        ← отдельный файл, camelCase латиницей
  index.ts             ← импорт и запись в массив poets[]
src/types/poet.ts      ← общий тип Poet
```

Правила:

- Один поэт — один файл. Не собирать библиотеку обратно в гигантский `poets.ts`.
- Имя файла: `sergeiYesenin.ts`, `annaAkhmatova.ts`.
- `id`: kebab-case, например `sergei-yesenin`.
- Порядок массива определяет владелец по эпохе и значению; не сортировать вслепую.
- Общие редакторские преобразования допустимы как временный этап аудита. Перед слиянием зрелый текст следует перенести в читаемые исходные данные, а лишние слои удалить.

## 2. Обязательные поля

| Поле | Стандарт |
|---|---|
| `id` | Уникальный kebab-case; становится маршрутом `/poets/<id>`. |
| `name` | Краткое имя с правильной орфографией и `ё`, где она нормативна. |
| `fullName` | Полное имя. |
| `birthYear`, `deathYear` | Проверенные годы; `deathYear` отсутствует только у живущего человека. |
| `nationality` | `Русский` / `Русская`. |
| `photo` | `/images/<translit>.jpg`; путь проверяется сборкой. |
| `shortBio` | 1–2 предложения, примерно 280–360 знаков. Конкретный портрет, не рекламный лозунг. |
| `fullBio` | Обычно 5–9 абзацев. Хронология, даты, произведения, отношения, кризисы, смерть и наследие. |
| `rating` | Редакционная оценка 0–10. |
| `tags` | 3–6 содержательных тегов. |
| `poems` | Не менее двух стихотворений с проверенным текстом. |
| `famousWorks` | 5–7 главных произведений. |

Сильно рекомендуются:

- `testimonies` — 5–9 свидетельств современников и исследователей;
- `historicalNote` — короткий контекст эпохи;
- `spiritualSearch` — религиозный путь и мировоззрение;
- `moralPortrait` — документированный нравственный портрет;
- `authorCommentary` — краткая итоговая ремарка, только если она добавляет новую мысль.

## 3. Три редакционных голоса

### `spiritualSearch`

Отвечает на вопросы: что человек думал о Боге, Христе, вере, церкви, философии и смерти; как это менялось; какие источники позволяют судить.

Тон — аналитический и спокойный. Религиозный образ в стихотворении не превращается автоматически в свидетельство личной веры. Православное воспитание, причастие, крестное знамение или знание литургии требуют контекста и не равны возрождению.

### `moralPortrait`

Показывает установленные поступки и устойчивые черты, которые школьный портрет обычно сглаживает: прелюбодеяние, алкоголизм, жестокость, дуэльную культуру, азартную зависимость, богохульство, служение репрессивному аппарату.

Порядок:

1. назвать документированный факт;
2. обозначить качество источника и неопределённость;
3. показать человеческий плод поступка;
4. дать точное нравственное название;
5. применить Писание только там, где оно проясняет, а не заменяет исследование.

Не смешивать разные категории удобными словами. Карточная расточительность не всегда сребролюбие; религиозная тоска не всегда вера; политическая казнь и смерть за убеждения сами по себе не являются христианским мученичеством. Мученик за Христа страдает именно за исповедание Христа и верность Ему.

### `authorCommentary`

Один короткий абзац после уже завершённого портрета. Он не пересказывает `moralPortrait`, не произносит второй приговор и не заканчивается обязательной эффектной формулой. Поле можно опустить.

## 4. Моральный реализм и мера

### Называем прямо

- связь с чужим супругом — прелюбодеянием;
- устойчивое пьянство — алкогольной зависимостью;
- хулу на Бога — богохульством;
- доказанное самоубийство — самоубийством;
- участие в органах насилия — точной должностью и сроком службы;
- сознательный религиозный отказ — словами самого человека.

### Не делаем

- не цитируем мат и непристойную физиологию;
- не пересказываем сексуальные сплетни без надёжного основания;
- не выводим всю биографию из одного стихотворения;
- не объявляем болезнь или смерть наглядным Божьим наказанием;
- не сводим причинность самоубийства ни к одной медицинской, психологической, социальной или духовной формуле;
- не исключаем нравственную ответственность и духовное направление из биографии под видом медицинской осторожности;
- не превращаем согласие участников греха в нравственное оправдание;
- не компенсируем прямую оценку искусственным списком заслуг;
- не придумываем покаяние в последний миг и не используем разбойника на кресте как оправдание сознательного промедления;
- не злорадствуем и не строим карикатуру.

Тон определяется не знаменитостью и не симпатией редактора, а силой свидетельств. Ахматова и Пастернак получают ту же меру проверки, что Есенин и Маяковский: установленное называется прямо, спорное маркируется, недоказанное удаляется.

## 5. Русский литературный стиль

Страница поэта должна звучать как биографическая проза, а не как отчёт нейросети.

- Начинайте абзац с даты, письма, поступка, сцены или строки, а не всегда с редакционного тезиса.
- Чередуйте длину предложений.
- Держите в абзаце одну главную мысль.
- Убирайте повторы `не только…, но и…`, `нельзя свести…, но нельзя и…`, `это не…, а…`.
- Не повторяйте в каждом разделе слова `идол`, `ложный спаситель`, `падшее сердце`.
- Предпочитайте глагол отвлечённому существительному: `разрушал`, а не `осуществлял разрушение`.
- Не объясняйте читателю второй раз то, что уже показала сильная цитата.
- Не пишите декоративных жестов, погоды и внутренних мыслей, которых нет в источнике.
- Читайте итоговый текст вслух. Ровный одинаковый ритм — признак необходимости новой правки.

Рабочий вопрос: если удалить жирное выделение и последнюю афористическую фразу, останется ли хороший русский абзац?

## 6. Стихотворения

- Текст сверяется минимум по двум независимым источникам, один из них A+.
- `id` уникален во всей библиотеке.
- `analysis` объясняет форму, образ, контекст и движение мысли без готового психологического диагноза.
- `biblicalPerspective` добавляется только при реальной связи с Писанием или ясной нравственной проблематике текста.
- Библейский комментарий цитирует те же слова, которые лежат в поле `text`.
- Литературный образ не считается стенограммой личной жизни.
- Сходство с библейским сюжетом не означает личной веры автора.
- `<...>` допустимо для честно обозначенного пропуска длинного текста.

## 7. Свидетельства

Цель — 5–9 записей разных типов.

```ts
{
  author: '<Ф.И.О.>',
  role: '<отношение к поэту>',
  kind: 'contemporary', // либо 'historian'
  quote: '<точная цитата или явно помеченный пересказ>',
  source: '<издание и год>',
  sourceUrl: '<проверенная ссылка>',
}
```

Правила:

- современник близок к событию, но может лгать, враждовать или оправдываться;
- поздний мемуар не становится стенограммой;
- пересказ помечается словом `(пересказ)`;
- музейный сотрудник не получает автоматически статус нейтрального историка;
- мёртвая либо ведущая не на тот текст ссылка хуже отсутствующей;
- количество повторов одной легенды не равно количеству независимых свидетельств.

## 8. Иерархия источников

### A+ — первичное свидетельство

Автографы, письма, дневники, протоколы, официальные и архивные документы, академические ПСС, первые публикации, ФЭБ, РВБ, Библиотека Поэта с текстологическим аппаратом.

### A — независимое исследование

Академическая монография, критическая биография, статья ИМЛИ/Пушкинского Дома, документальное исследование, реабилитационный или судебный материал. Источник показывает документы, отделяет гипотезу и позволяет проверить ссылку.

### B — институциональный и популярный материал

Музеи, мемориальные дома, выставочные тексты, `culture.ru`, ЖЗЛ, качественная журналистика. Они могут дать редкий документ, но комментарий учреждения остаётся интерпретацией и проверяется на конфликт интересов.

### C — поисковый след

Энциклопедический пересказ без ссылок, популярный блог, видеоэссе, социальная сеть. Используется для поиска версии, но не как доказательство спорного факта.

Каждая спорная дата и канонический текст проверяются минимум по двум независимым источникам. Несколько страниц, перепечатавших один мемуар, считаются одним источником.

## 9. Чек-лист перед публикацией

```text
[ ] Файл импортирован и добавлен в poets[]
[ ] Все обязательные поля заполнены
[ ] id и poem-id уникальны
[ ] Даты проверены и образуют непротиворечивую хронологию
[ ] Каждый стих сверен по 2+ источникам, один A+
[ ] Свидетельства имеют источник и корректную маркировку
[ ] spiritualSearch, moralPortrait и authorCommentary не дублируются
[ ] Сильные факты основаны на A+/A
[ ] Спорные сцены не выданы за установленные
[ ] Явный грех назван без эвфемизма
[ ] Библейские ссылки точны и не заменяют анализ
[ ] Текст прочитан вслух и не звучит машинно
[ ] Изображение существует по указанному пути либо отсутствие сознательно обработано
[ ] Пройдены validate-library, validate-essays, validate-literary-style, typecheck и build
[ ] В PR описаны источники, исправленные легенды и оставшиеся неопределённости
```

## 10. Краткий шаблон

```ts
import { Poet } from '../../types/poet';

export const <nameVar>: Poet = {
  id: '<kebab-id>',
  name: '<Имя Фамилия>',
  fullName: '<Имя Отчество Фамилия>',
  birthYear: <YYYY>,
  deathYear: <YYYY>,
  nationality: 'Русский',
  photo: '/images/<translit>.jpg',
  shortBio: '<1–2 конкретных предложения>',
  fullBio: `<Хронологическая биография с пустой строкой между абзацами>`,
  rating: <0–10>,
  tags: ['<Эпоха>', '<Течение>', '<Тема>'],
  poems: [
    {
      id: '<poet>-1',
      title: '<Название>',
      year: <YYYY>,
      text: `<проверенный текст>`,
      analysis: '<литературный разбор>',
      // biblicalPerspective — только при органичной связи
      rating: <0–10>,
    },
  ],
  articles: [],
  historicalNote: '<контекст эпохи>',
  spiritualSearch: '<религиозный путь и мировоззрение>',
  moralPortrait: `<документированный нравственный портрет>`,
  authorCommentary: '<необязательная короткая ремарка>',
  testimonies: [],
  famousWorks: ['<Произведение>'],
};
```
