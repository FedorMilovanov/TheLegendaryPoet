# POET_AUTHORING_GUIDE.md — гид по добавлению и унификации поэтов

Последний раз обновлено: 2026-07-16. Владелец: Фёдор Милованов.

Это **единый стандарт** для введения новых поэтов в THE LEGENDARY POET. Цель —
чтобы каждый поэт, от первого до сотого, вводился одинаково, в едином стиле, без
хаотичных «авторских» отступлений. Если ты (человек или агент) добавляешь поэта —
этот документ obligаторный; отступления допустимы только по осознанной причине,
зафиксированной в PR.

> Смежные документы:
> - `ARENA_PROJECT_PROTOCOL.md` — архитектура и дизайн.
> - `src/docs/THEOLOGICAL_GUIDELINES.md` — богословская оптика и моральный реализм.
> - `PROJECT_CHARTER.md` — кто владелец, какая позиция, зачем проект.
> - `AGENT_ANTI_REGRESSION_PROTOCOL.md` — защита от регрессий и гейт.

## 0. Кто и зачем: редакционная позиция (в одном абзаце)

Проект ведёт **Фёдор Милованов** (gospod-bog.ru), позиция — **историческое
евангельское христианство**: богодухновенность и достаточность Писания, спасение
по благодати через веру во Христа, грамматико-историческая герменевтика (в духе
Макартура, Спраула, Сперджена). **Мы не православные и не секуляры.** Это значит:
когда мы говорим о духовной стороне — опираемся на Писание, а не на предание; не
обожествляем поэтов и не баптизируем их задним числом; даём честную моральную
оценку их жизни, включая грехи, о которых молчит школа. Подробно —
`src/docs/THEOLOGICAL_GUIDELINES.md`.

## 1. Архитектура: один поэт — один модуль

```
src/data/library/
  <poetName>.ts        ← НОВЫЙ ФАЙЛ (camelCase латиницей: daniilAndreev.ts)
  index.ts             ← добавь импорт + в массив poets[]
src/types/poet.ts      ← тип Poet (правь только при реальной необходимости)
```

**Правила:**
- Один поэт = один файл. Никогда не сливать обратно в гигантский `poets.ts`.
- Имя файла — `latinCamelCase` по фамилии: `sergeiYesenin.ts`, `annaAkhmatova.ts`.
- В `index.ts` — добавить импорт первой строкой группы и запись в массив `poets`.
- `poets.ts` остаётся тонким агрегатором (`export { poets, articles, musicTracks }`).
- Порядок в массиве `poets` — **по эпохе/важности**, его фиксирует владелец; не
  сортируйте алфавитно вслепую.

## 2. Обязательные поля (без них — не публиковать)

| Поле | Тип | Назначение и стандарт |
|---|---|---|
| `id` | string | kebab-case латиницей: `'alexander-pushkin'`. Уникален. Становится URL `/poets/<id>`. |
| `name` | string | Краткое имя: `'Александр Пушкин'`. |
| `fullName` | string | Полное: `'Александр Сергеевич Пушкин'`. |
| `birthYear` / `deathYear` | number | Год(ы). `deathYear` обязателен для ушедших; `undefined` только для живых. |
| `nationality` | string | `'Русский'` / `'Русская'`. |
| `photo` | string | `/images/<translit>.jpg` (60–200 КБ, квадрат или портрет, dignified). |
| `shortBio` | string | **1–2 предложения, ≈280–360 знаков.** Эпиграмматичный, не умильный портрет. Идёт в карточку и SEO. |
| `fullBio` | string | **5–9 абзацев** (раздел — пустая строка `\n\n`). Жизнь по этапам; факты с датами; без выдумки. |
| `rating` | number | 0–10, оценка редакции. Для канона 9.4–10, для рядовых ниже. |
| `tags` | string[] | 3–6 тегов: эпоха/течение/темы (`'Золотой век'`, `'Символизм'`). |
| `poems` | Poem[] | **Минимум 2** (лучше 3–4). Канонический текст сверен по 2+ источникам. |
| `famousWorks` | string[] | 5–7 главных произведений (названия книг/поэм/стихотворений). |

## 3. Сильнo рекомендуемые поля

| Поле | Назначение |
|---|---|
| `testimonies` | Свидетельства современников + оценки историков. **Цель 5–9 записей**, у каждой — `author`, `role`, `kind`, `quote`, `source` (книга+год), желательно `sourceUrl`. |
| `historicalNote` | 2–4 предложения об эпохе (войны, режим, культурный фон). |
| `spiritualSearch` | **Духовный путь и мировоззрение** — трезвая фиксация исканий, без «баптизирования». |
| `moralPortrait` | **Моральный портрет** — честная оценка грехов и падений (см. §5). |

## 4. Опциональные поля

`epoch`, `epochLabel`, `initials`, `coverImage`, `music`, `articles`, `links`
(youtube/vk/rutube/website), `authorCommentary` (короткая итоговая ремарка).

> `spiritualSearch`, `moralPortrait`, `authorCommentary` — **три разных голоса**, не
> дублируйте их. Разграничение — в §5 и §6.

## 5. Три голоса редакции: как их не путать

Это **самое важное правило стиля**, нарушение которого = регресс.

### `spiritualSearch` — «Духовный путь и мировоззрение»
- **О чём:** чего поэт *искал* в религии/философии, как менялось его мировоззрение,
  влияния (Соловьёв, Шопенгауэр, марксизм), колебания веры.
- **Тон:** аналитический, трезвый, констатирующий.
- **Чего не делать:** не превращать в исповедь, не делать поэта верующим без
  оснований, не морализаторствовать (это для moralPortrait).

### `moralPortrait` — «Моральный портрет»
- **О чём:** задокументированные **грехи и разрушительные черты** жизни —
  прелюбодеяние, пьянство, дуэльный азарт, богоборчество, жестокость, сребролюбие.
  То, о чём молчит хрестоматийный «светлый» портрет.
- **Тон:** прямой, библейски-реалистичный, без умиления и без злорадства. Грех
  называется по имени и получает ссылку на Писание (книга/глава/стих).
- **Цензура:** см. §6.
- **Когда поле нужно:** когда у поэта есть выраженная нравственная проблематика.
  Для почти безупречных жизней можно опустить или дать короткое суждение.

### `authorCommentary` — «Авторская ремарка»
- **Короткое** (1 абзац) итоговое слово автора проекта для вопиющих случаев
  (богоборчество, особая трагедия). НЕ пересказывает moralPortrait, а ставит
  точку. Если moralPortrait уже сказал главное — поле можно опустить.

## 6. Моральный реализм и цензура (отдельно — критично)

Проект говорит о грехах поэтов **открыто и с библейской оценкой**, потому что
молчать = лгать и создавать кумир. Но делается это с дисциплиной:

### Что ДЕЛАЕМ
- Называем грех по имени: прелюбодеяние, блуд, пьянство, сребролюбие, гордыня,
  богоборчество, жестокость, дуэльный азарт.
- Даём библейскую ссылку (Еф. 5:18; 1 Кор. 6:9–10; Евр. 13:4; Гал. 5:19–21;
  Исх. 20:14 и т.д.) — точно, книга/глава/стих.
- Указываем, **откуда** факт известен (мемуары, документы, собственные стихи).
- Сохраняем уважение к таланту и сострадание к человеческой трагедии.
- Признаём неопределённость там, где данных мало.

### Чего НЕ ДЕЛАЕМ
- ❌ **Не воспроизводим непечатную брань и мат.** О вольной/похабной лирике
  говорится («известны непристойные эпиграммы», «циничные застольные стихи»), но
  сами слова/строки **не цитируются**.
- ❌ **Не описываем половые акты и непристойные детали.** «Связь», «прелюбодеяние»,
  «двойная жизнь» — да; физиология — нет.
- ❌ Не пересказываем сплетен и недоказанных обвинений (особенно в сексуальной
  сфере — клевета на мёртвых недопустима). Нет двух источников — не пишем.
- ❌ Не злорадствуем, не пользуемся грехом для «жареного».
- ❌ Не вставляем moralPortrait и библейские оценки **везде и механически**. Они
  ставятся там, где есть реальное основание и где это проясняет портрет.

### Границы поэтов (примеры калибровки)
- **Пушкин:** донжуанский список, долги, дуэли, непристойные эпиграммы — даём.
- **Тютчев:** 14-летнее прелюбодеяние с Денисьевой, трое внебрачных детей,
  циничный царедворец — даём.
- **Маяковский:** сожительство с чужой женой, богохульство — даём.
- **Есенин:** алкоголизм, пьяное рукоприкладство, богоборчество — даём.
- **Лермонтов:** дуэльный азарт, желчность — даём сдержанно.
- **Гумилёв:** прелюбодеяние в браке с Ахматовой, юношеские попытки
  самоубийства — даём, но с явным признанием его мужества и достоинства перед
  казнью (самый «светлый» из портретов, потому и самая строгая мерка).
- **Ахматова / Пастернак:** моральная проблематика есть, но деликатнее; поле
  короче и бережнее.

> **Принцип калибровки (важно):** тон портрета соразмеряется с тяжестью жизни.
> Оголённое распутство (Есенин, Маяковский) требует жёсткой библейской оценки;
> деликатные нравственные трещины (Ахматова, Пастернак) — бережной констатации,
> без осуждения и без замалчивания. «Светлые» портреты (Гумилёв) — самые строгие:
> кому больше дано, с того больше и спросится (Лк. 12:48).

## 7. Стихи (`poems[]`): стандарт канонического текста

- **Только канонический текст**, сверен по ≥2 источникам, хотя бы один A+
  (ФЭБ, РВБ, академическая Викитека, Библиотека Поэта со ссылкой на автограф).
- Поля: `id` (уникален во всей библиотеке!), `title`, `text`, `year`, `rating`.
- `analysis` — краткий литературный разбор (стиль, образ, контекст).
- `biblicalPerspective` — **только если в стихе есть реальный библейский образ**.
  Называем место Писания точно и объясняем, откуда образ и как поэт его преломил.
  **Никогда** не превращаем стих в доказательство веры автора. И — критично —
  `biblicalPerspective` обязан **цитировать те же слова, что лежат в `text`**
  (именно так в 2026-07 всплыла ошибка в «Наш век» Тютчева).
- `<...>` допустимо для пропуска строф в длинных стихах (особенно в эссе).
- `mood` — опциональная эмоциональная палитра.

---

## 8. Свидетельства (`testimonies[]`)

- Цель **5–9 записей**: микс `kind: 'contemporary'` (жена/друг/врач) и
  `kind: 'historian'` (литературовед/биограф).
- `quote` — вербатим; если пересказ — явно пометить `(пересказ)`.
- `source` — книга/мемуары/статья + год. `sourceUrl` — желательно (мёртвые ссылки
  хуже отсутствующих — проверяй перед вставкой).
- **Не включать** спорные/неподтверждённые атрибуции и легенды (последние слова
  Гумилёва и т.п.) — см. `docs/RESEARCH_SOURCES.md`, методологическую оговорку.

## 9. Чек-лист перед публикацией нового поэта

```
[ ] Файл создан, импортирован и добавлен в массив poets[] в index.ts
[ ] Все обязательные поля заполнены (§2)
[ ] birthYear/deathYear корректны, deathYear > birthYear
[ ] id и poem-id уникальны (нет коллизий со всей библиотекой)
[ ] Каждый текст стиха сверен по ≥2 источникам (≥1 A+)
[ ] testimonies: ≥5 записей, у каждой источник; нет неподтверждённых легенд
[ ] spiritualSearch / moralPortrait / authorCommentary НЕ дублируют друг друга (§5)
[ ] moralPortrait соблюдает цензуру §6 (нет мата, нет физиологии, есть источники)
[ ] biblicalPerspective цитирует те же слова, что text; ссылки точны
[ ] Фото добавлено в public/images/ и путь совпадает с photo
[ ] Прогнан гейт: npx tsx scripts/validate-library.ts (0 ERROR) + tsc + build
[ ] В PR указаны источники сверки (≥30 ссылок для серьёзного поэта — цель проекта)
```

## 11. Иерархия источников (цитата из VERIFICATION_REPORT, кратко)

- **A+ — первоисточник:** автограф / академическое ПСС, ФЭБ, РВБ, Библиотека Поэта
  со ссылкой на автограф, первые публикации.
- **A — авторитетный вторичный:** академическая Викитека, culture.ru, именные
  сайты-музеи, монографии/биографии ЖЗЛ, официальные документы.
- **B — крайний случай:** качественная журналистика со ссылками на архивы.

**Правило:** каждый канонический текст стиха и каждая спорная дата — минимум по
двум независимым источникам, хотя бы один A+. Сверка только по «Википедии» = регресс.
